## Scope

This policy covers the Resonance repository and any future artifacts that ship as part of the project.

## Reporting a Vulnerability

If you discover a security issue, do not open a public issue or discussion.
Report it through the project's maintainer-managed private channel or the agreed security contact path.

Include:

- A clear description of the issue.
- The affected component or document.
- Reproduction steps, if applicable.
- Any mitigation ideas you already have.

## Disclosure Principles

- Keep reports confidential until the maintainers coordinate a release or advisory.
- Minimize unnecessary details in public threads.
- Treat security fixes as time-sensitive and review them promptly.

## Current Support State

The project is currently in foundation mode and does not yet ship a production implementation.
Security guidance will be expanded once executable components exist.
