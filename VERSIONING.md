## Policy

Resonance will follow Semantic Versioning, with the understanding that the 0.x series is still a foundation phase.

## Pre-1.0 Meaning

- `0.1` establishes structure, terminology, and contributor workflow.
- Later `0.x` releases may adjust contracts while the public API is still being stabilized.
- Public-facing compatibility promises should be stated explicitly in release notes and ADRs.

## Compatibility Notes

- Breaking changes should be called out clearly.
- Stable concepts should be isolated early so they can survive future implementation growth.
