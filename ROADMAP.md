## Foundation Phase

- Confirm project scope, principles, and non-goals.
- Maintain architecture documents and ADR structure.
- Establish build, test, lint, and release scaffolding.
- Keep the repository engine-agnostic and implementation-free.

## Stabilization Phase

- Define stable core contracts.
- Draft protocol boundaries without shipping runtime behavior.
- Prepare compatibility rules for SDKs and future versions.

## Integration Phase

- Add SDK entry points for supported runtimes.
- Introduce reference examples and validation scenarios.
- Evolve automation around quality gates and release readiness.

## Ecosystem Phase

- Expand documentation and public guidance.
- Add benchmarks, compatibility notes, and migration support.
- Prepare long-term governance for a broad contributor base.
