## Module Overview

- `core/` domain boundary and shared concepts.
- `protocol/` future protocol definitions and contracts.
- `server/` optional server-side components.
- `sdk/` language and engine integrations.
- `architecture/` structural guidance.
- `decisions/` ADR history.

## Ownership Model

Each module should have a clear owner, a narrow purpose, and a documented dependency direction.
