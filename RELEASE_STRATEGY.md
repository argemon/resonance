## Purpose

Release strategy defines how Resonance moves from foundation work to stable public delivery.

## Release Model

- Releases should be deliberate and documented.
- Release notes should explain scope, compatibility, and migration impact.
- Significant changes should be tied back to ADRs and versioning guidance.

## Pre-release Discipline

- Do not ship unfinished conceptual boundaries as if they were stable contracts.
- Keep early versions explicit about their maturity.
- Prefer short release cycles during the 0.x phase.

## Current Status

No release process is active yet because the repository does not contain production code.
